# UMX 首页样式优化方案

> 基于 2026-02-07 线上版本分析，遵循移动优先原则

---

## 一、总体评价

当前首页已具备完整的功能结构和基础设计系统，金色品牌色运用一致，组件化程度高。但从**视觉节奏、信息密度、氛围感、精致度**四个维度来看，仍有明显提升空间。以下按区域逐一分析。

---

## 二、逐区域优化意见

### 1. Header 导航栏

**现状问题：**
- 背景纯白，与页面其他区域缺乏层次区分
- Logo 区域 `umx.art` 字体普通（系统字体），缺少品牌辨识度
- 搜索框胶囊形态中规中矩，灰色背景与金色主题脱节
- 通知图标触控区域偏小（32x32），不符合 44px 最小触控区建议

**优化建议：**
- [ ] **Header 增加微妙的底部分割线或阴影**：使用 `box-shadow: 0 1px 0 rgba(0,0,0,0.04)` 替代硬边框，在滚动时增加 `backdrop-filter: blur(12px)` 毛玻璃效果
- [ ] **搜索框加入品牌色暗示**：placeholder 左侧搜索图标改为金色渐变色，搜索框边框使用极淡金色 `border: 1px solid rgba(212,175,55,0.08)`
- [ ] **通知图标触控区扩大至 44x44**，保持视觉大小不变（padding 补偿）
- [ ] **Logo 文字加入品牌渐变**：`umx.art` 使用金色渐变 `background-clip: text`，增强品牌记忆

---

### 2. 轮播 Banner

**现状问题：**
- 圆角 `16px` 与整体页面其他卡片圆角一致，缺少主视觉的特殊感
- 底部指示点太小（4px），在金色渐变背景上辨识度弱
- Banner 区域缺少内容层：只有图片，没有文字叠加信息（标题/副标题/CTA按钮），信息承载力弱
- 左右两侧的 `12px` 边距使 Banner 看起来像卡片而非沉浸式主视觉

**优化建议：**
- [ ] **考虑 Banner 全宽模式**：去掉两侧 margin，让 Banner 占满宽度，提升首屏冲击力。圆角仅保留底部 `border-radius: 0 0 20px 20px`
- [ ] **增加 Banner 文字叠加层**：在渐变遮罩上叠加活动标题 + 简短描述，提升信息密度
- [ ] **指示点尺寸微调**：非活跃点从 4px 提升至 6px，活跃状态宽度从 14px 提升至 18px，增加可读性
- [ ] **添加自动轮播进度条**：在 Banner 底部添加极细的金色进度条（height: 2px），替代或补充圆点指示器，更现代

---

### 3. 系统通知条

**现状问题：**
- 暖黄色背景 `#FFF9E6` 视觉上与金色主题协调，但样式过于朴素
- 高度仅 32px，在整体布局中存在感弱
- 缺少进入/退出动画，切换突兀

**优化建议：**
- [ ] **增加左侧图标动画**：小喇叭图标添加轻微摇摆动画（`@keyframes shake`），吸引注意力
- [ ] **切换动画优化**：当前 `slide-up` 仅做了位移+透明度，建议新文字从下方滑入（`slide-in-up`），形成"滚动公告"的完整感
- [ ] **背景增加微渐变动画**：使用 `background-size: 200% 200%` + `animation` 实现背景色缓慢流动效果

---

### 4. 功能宫格 (FeatureGrid)

**现状问题：**
- 7 个入口横向排列，但宽度按 `100%/5` 计算，多出的 2 个需要滚动才能看到——**用户可能根本不知道可以滑动**
- 图标背景色使用行内样式 15% 透明度，颜色饱和度低，视觉层次弱
- 无横向滚动指示器，违反"可见性"启发式原则
- 图标文字 11px 偏小，高龄用户阅读困难

**优化建议：**
- [ ] **添加滚动渐变遮罩**：右侧添加 `linear-gradient(to left, white 0%, transparent 100%)` 遮罩，暗示有更多内容
- [ ] **图标背景饱和度提升**：透明度从 15% 提升至 20%-25%，或使用双层渐变增加深度
- [ ] **首屏显示 5 个，第 6 个露出一半**：调整 item 宽度从 `100%/5` 改为 `calc((100% - 32px) / 4.5)`，让第 5 个半露出来，暗示可滑动
- [ ] **文字字号提升至 12px**：从 11px 提升到 12px，改善可读性

---

### 5. 热卖榜 + 活动角 (HotRankAndActivity)

**现状问题：**
- 这是首页最复杂的信息区块，但视觉层级与上下区块区分不足
- 左右栏 55:45 比例在小屏上导致活动角图标过于拥挤
- 排名数字（1、2、3）用了不同颜色但无背景，视觉冲击力不够
- "热卖榜"标题与列表间距 10px 偏紧
- 活动角的 2x2 宫格图标 40x40 偏小，文字 11px 可读性差
- 两个卡片的 `box-shadow: var(--shadow-xs)` 太轻，几乎不可见

**优化建议：**
- [ ] **增强卡片存在感**：阴影从 `--shadow-xs` 升级为 `--shadow-sm`，增加 `border: 1px solid rgba(0,0,0,0.04)` 微边框
- [ ] **排名数字添加背景圆**：1/2/3 名添加 20x20 的圆形背景（金/银/铜色），数字白色居中
- [ ] **热卖榜增加"查看全部"入口**：标题右侧添加 `>` 箭头导航
- [ ] **活动角图标增大至 44x44**：满足触控友好标准
- [ ] **两栏比例调整为 1:1**：在 375px 宽度下均分空间，减少拥挤感
- [ ] **标题行增加底部分割线**：使用极淡的 `border-bottom: 1px solid var(--color-border)` 分隔标题与内容

---

### 6. Tab 切换区 (UFilterTabs - underline 模式)

**现状问题：**
- 下划线指示器直接使用 `--color-primary` 纯色，缺少品牌感
- Tab 间距 24px 过于均匀，首个 Tab 与左侧页面边距不一致（16px padding vs 24px gap）
- 非选中态文字 `--color-text-tertiary` (#9CA3AF) 偏淡，对比度不足
- Tab 切换无过渡动画，下划线无滑动效果

**优化建议：**
- [ ] **下划线改为渐变色**：使用 `--gradient-gold` 替代纯色，增加视觉精致度
- [ ] **添加下划线滑动动画**：Tab 切换时下划线平滑滑动到新位置（使用 `transform: translateX()` + `transition`）
- [ ] **非选中态文字颜色加深**：从 `#9CA3AF` 改为 `#6B7280`（--color-text-secondary），提高可读性
- [ ] **Tab 区域增加 sticky 定位**：下划线 Tab 在滚动时吸顶（`position: sticky; top: 56px`），方便快速切换
- [ ] **首个 Tab 左边距对齐页面 padding**：确保 `padding-left: 16px` 与页面整体一致

---

### 7. 藏品卡片网格 (CollectionCard - home variant)

**现状问题：**
- 图片高度固定 165px，导致横版图片上下出现大面积空白（被裁切），竖版图片又被过度压缩
- 卡片内容区信息层级不清晰：标题、作者、发行方式、发行份数四个信息在 2 行内挤在一起
- "合成"/"直卖"标签与"发行份数"并排，信息优先级不明确
- 作者头像 20x20 太小，用户默认灰色头像缺乏个性
- 卡片间距 14px 配合 `padding: 0 16px`，实际有效展示面积偏小
- 卡片底部缺少明确的 CTA 引导（如"查看详情"或价格信息）
- **长标题被截断，如"冯戈数字肖像定制资格卡——链上丹青计划"只显示"冯戈数..."**

**优化建议：**
- [ ] **图片容器改为固定宽高比**：使用 `aspect-ratio: 3/4` 或 `4/3` 替代固定高度，适配不同图片比例
- [ ] **卡片标题改为 2 行截断**：从 `white-space: nowrap` 改为 `-webkit-line-clamp: 2`，展示更多标题信息
- [ ] **发行方式标签移至图片区域左下角**：使用半透明背景覆盖在图片上（类似 Netflix 标签风格），释放内容区空间
- [ ] **作者信息区视觉增强**：头像从 20px 提升至 24px，添加微边框；作者名 max-width 从 50px 放宽至 60px
- [ ] **卡片间距微调**：gap 从 14px 降至 12px，左右 padding 从 16px 降至 12px，增加图片展示面积
- [ ] **增加价格信息**：即使首页 variant 不显示价格，也应显示一个"起拍价"或"市场价"的信息锚点
- [ ] **添加卡片进场动画**：使用 `IntersectionObserver` 触发淡入上滑动画（`opacity: 0 → 1`, `translateY(20px) → 0`），页面更有生命力

---

### 8. 底部 TabBar

**现状问题：**
- 仅 3 个 Tab（首页、市场、个人中心），在 375px 宽度上间距过大，看起来空旷
- 图标与文字间距 4px 偏紧
- 激活态仅文字加粗（font-weight: 400→500），变化不明显，缺乏视觉反馈
- 缺少品牌色运用，纯灰色图标 → 灰色加粗的转变太微妙

**优化建议：**
- [ ] **激活态引入品牌色**：选中 Tab 的图标和文字使用 `--color-primary` 金色
- [ ] **添加选中指示器**：图标上方或下方添加 3px 高的金色圆点或短线条
- [ ] **图标激活态微动画**：选中时图标 `transform: scale(1.1)` + `transition: 0.2s`
- [ ] **考虑增加 Tab 项**：3 个 Tab 过少，建议至少 4 个（如增加"热卖"或"发现"）
- [ ] **安全区域处理**：确认 `env(safe-area-inset-bottom)` 在 iPhone X+ 设备上正确工作

---

### 9. 页面整体节奏与氛围

**现状问题：**
- 区块之间的间距节奏不统一：Banner→通知条 10px，通知条→宫格 4px，宫格→热卖榜 12px，热卖榜→Tab 8px
- 页面整体偏"白"和"平"，缺少纵深感和视觉节奏变化
- 顶部装饰性渐变（金色+紫色）效果太弱（opacity 0.08/0.04），几乎不可见
- 各个区块之间缺少明确的视觉分割
- 滚动体验线性，缺乏"呼吸感"

**优化建议：**
- [ ] **统一区块间距为 16px**：所有一级区块（Banner、通知、宫格、热卖、Tab、卡片）之间统一使用 `--section-gap: 16px`
- [ ] **增强顶部氛围渐变**：金色渐变透明度从 0.08 提升至 0.12-0.15，高度从 320px 缩短至 240px 使效果更集中
- [ ] **在 Tab 区域上方添加分割元素**：使用微渐变线条 `border-top: 1px solid rgba(0,0,0,0.06)` 或一行"为你推荐"文案区分运营区和内容区
- [ ] **藏品区域背景微调**：卡片网格区域使用 `--color-bg-secondary` (#F9FAFB) 作为背景，与上方运营区形成色差
- [ ] **页面底部添加"到底了"提示**：简单的文案 + 分割线 + 回到顶部按钮

## 三、性能相关优化

| 问题 | 建议 |
|------|------|
| Banner 图片未做响应式加载 | 使用 `srcset` 或根据 DPR 加载 1x/2x/3x 图片 |
| 藏品卡片全量渲染 | 使用虚拟列表（如 `react-window`）或分页加载 |
| 顶部装饰渐变使用 `position: fixed` | 改为 `position: absolute` 减少重绘开销 |
| 无骨架屏 | 首屏加载时展示骨架屏，提升感知速度 |

## 四、优先级排序

### P0 - 高影响、低成本（立即可做）
1. TabBar 激活态加入品牌色
2. 卡片标题改为 2 行截断
3. 非选中 Tab 文字颜色加深
4. 统一区块间距
5. 增强顶部氛围渐变
6. 通知图标触控区扩大

### P1 - 高影响、中成本（本周）
1. Banner 全宽模式改造
2. Tab 下划线滑动动画
3. Tab 区域 sticky 吸顶
4. 热卖榜/活动角卡片增强
5. 功能宫格滚动提示
6. 卡片图片改为固定比例

### P2 - 中影响、高成本（后续迭代）
1. 卡片进场动画
2. Banner 文字叠加层
3. 自动轮播进度条
4. Header 毛玻璃效果
5. 虚拟列表性能优化
6. 骨架屏实现
